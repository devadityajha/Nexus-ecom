# 🚧 Nexus-Ecom | Advanced Logic Sandbox (Work in Progress)

> **Status:** 🏗️ Actively Building | Phase 1 (Backend Architecture)

Welcome to my personal sandbox! This project is currently under active development. I am building this e-commerce engine not as a basic clone, but as a dedicated practice ground for mastering complex backend architecture, dynamic data flow, and secure authentication.

Because this is a learning-first project, the tech stack and feature list will evolve as I tackle new engineering challenges.

---

## 🎯 The Goal of This Project

Instead of rushing to build a pretty UI, my focus right now is on the **Logic Layer**:

- Structuring a clean, scalable Express backend.

- Managing complex database queries without sacrificing performance.

- Implementing production-grade security patterns.

---

## 🛠️ Current Tech Stack (So Far)

The stack is minimal right now as I build out the core foundation. More tools will be added as the project scales.

- **Core:** Node.js, Express.js, MongoDB (Mongoose)

- **Frontend:** React.js

- **Testing:** Postman

- **Up Next (TBD):** State Management (Zustand/Redux?), UI Styling, etc.

---

## 🚀 What's Working Right Now

Here is what I have successfully implemented and tested so far:

- **MVC Architecture:** Clean separation of routes, controllers, and models.

- **Product Engine:** Full CRUD operations for the Product model.

- **Dynamic Query Foundation:** Initial setup for handling URL queries via backend controllers.

---

## 🧪 What I'm Currently Engineering (WIP)

These are the advanced features I am actively coding and experimenting with right now:

- **Scalable Data Streaming:** Implementing **Offset & Page Pagination** to handle large datasets efficiently.

- **Advanced Querying:** Building the logic for simultaneous Searching, Multi-checkbox Filtering (using MongoDB `$in`), and Sorting.

- **Dual-Token Authentication:** Designing a secure login flow using short-lived Access Tokens and long-lived Refresh Tokens stored in HTTP-only cookies.
